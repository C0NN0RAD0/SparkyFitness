# Proxmox Installation Guide

This guide covers how to install SparkyFitness on Proxmox VE in an LXC container using the included install script.

## Quick Install

Run the following command in the **Proxmox VE shell** to create a new LXC container and install SparkyFitness from this repository:

```bash
bash -c "$(wget -qLO - https://raw.githubusercontent.com/C0NN0RAD0/SparkyFitness/main/install/proxmox.sh)"
```

This script:
- Creates a Debian 13 LXC container (2 CPU cores, 2 GB RAM, 4 GB disk by default)
- Installs Node.js, pnpm, nginx, and PostgreSQL inside the container
- Builds and deploys the SparkyFitness frontend and backend
- Configures and starts all required services automatically

Once complete, SparkyFitness will be accessible at `http://<container-ip>`.

## Updating

To update an existing installation to the latest release from this repository, run the same command again from the Proxmox shell and choose **Update** when prompted:

```bash
bash -c "$(wget -qLO - https://raw.githubusercontent.com/C0NN0RAD0/SparkyFitness/main/install/proxmox.sh)"
```

The update script will:
1. Stop services
2. Back up your uploaded files and database backups
3. Download and deploy the latest release from this repository
4. Reinstall dependencies and rebuild the frontend
5. Restore backed-up data and restart services

> **Note:** The update command fetches the latest **GitHub Release** from `C0NN0RAD0/SparkyFitness`. Make sure a release exists in the repository before running an update.

## Testing from a Branch / PR

If you want to test an unreleased fix or feature from a specific branch (for example, a pull request that has not yet been merged), you can use the `SPARKYFITNESS_BRANCH` environment variable with the **update path** of the install script.

> **Prerequisites:** You need an existing SparkyFitness installation in the container. If you don't have one yet, run the normal install first and let it complete as far as it can. The container will still be usable as a base even if the frontend build step failed.

**Step 1 – Run the install script from the branch you want to test**, setting `SPARKYFITNESS_BRANCH` to the branch name:

```bash
SPARKYFITNESS_BRANCH=<branch-name> \
  bash -c "$(wget -qLO - https://raw.githubusercontent.com/C0NN0RAD0/SparkyFitness/<branch-name>/install/proxmox.sh)"
```

Replace `<branch-name>` with the actual branch name. For example, to test the branch that fixes the Proxmox LXC build (`copilot/create-sparkyfitness-lxc-again`):

```bash
SPARKYFITNESS_BRANCH=copilot/create-sparkyfitness-lxc-again \
  bash -c "$(wget -qLO - https://raw.githubusercontent.com/C0NN0RAD0/SparkyFitness/copilot/create-sparkyfitness-lxc-again/install/proxmox.sh)"
```

When prompted, choose **Update**.

**What this does:**
- Fetches the `install/proxmox.sh` script directly from the specified branch.
- Instead of downloading the latest GitHub Release tarball, it downloads the source code of that branch directly from GitHub.
- Rebuilds and redeploys the frontend and backend from that branch's code.
- Your database and uploaded files are backed up and restored automatically.

> **Note:** The `SPARKYFITNESS_BRANCH` option only affects the **update path**. Fresh installs (first-time container creation) still use the community-scripts framework, which deploys from the latest GitHub Release. For a completely fresh test, run the normal install first and then use the update path as described above.
